# 版本標籤系統

## 標籤說明

### 功能標籤
- `v1.0.0` - 初始版本
- `v1.1.0` - 新功能添加
- `v1.1.1` - 錯誤修復
- `v2.0.0` - 重大更新

### 狀態標籤
- `stable` - 穩定版本
- `beta` - 測試版本
- `alpha` - 開發版本
- `hotfix` - 緊急修復

### 類型標籤
- `feature` - 新功能
- `bugfix` - 錯誤修復
- `improvement` - 改進
- `breaking` - 重大變更
- `security` - 安全更新

## 版本命名規則

### 語義化版本 (Semantic Versioning)

格式：`主版本.次版本.修訂版本`

#### 主版本 (Major)
- 不兼容的 API 變更
- 重大功能更新
- 架構重構

#### 次版本 (Minor)
- 向後兼容的功能新增
- 新功能添加
- 重要改進

#### 修訂版本 (Patch)
- 向後兼容的錯誤修復
- 小改進
- 文檔更新

### 範例

```
v1.0.0    - 初始發布
v1.0.1    - 修復錯誤
v1.1.0    - 添加新功能
v1.1.1    - 修復新功能的錯誤
v2.0.0    - 重大更新，可能不兼容
v2.0.1    - 修復重大更新的錯誤
v2.1.0    - 在 v2.0.0 基礎上添加新功能
```

## 版本更新流程

### 1. 開發階段
```bash
# 創建開發分支
git checkout -b feature/新功能名稱

# 開發完成後
git add .
git commit -m "feat: 添加新功能"
```

### 2. 測試階段
```bash
# 創建測試版本
git tag v1.1.0-beta
git push origin v1.1.0-beta
```

### 3. 發布階段
```bash
# 創建正式版本
git tag v1.1.0
git push origin v1.1.0

# 更新版本文件
echo "v1.1.0" > VERSION
```

## 版本記錄格式

### 更新日誌格式
```markdown
## [版本號] - 日期

### 新增 (Added)
- 新功能描述

### 改進 (Changed)
- 改進描述

### 修復 (Fixed)
- 修復描述

### 移除 (Removed)
- 移除功能描述

### 安全 (Security)
- 安全更新描述
```

### 提交訊息格式
```
類型(範圍): 簡短描述

詳細描述（可選）

相關問題: #123
```

#### 類型說明
- `feat`: 新功能
- `fix`: 錯誤修復
- `docs`: 文檔更新
- `style`: 代碼格式
- `refactor`: 重構
- `test`: 測試
- `chore`: 構建過程或輔助工具的變動
